# Reflection
## Decisions
One of the most important decisions I made during this project was choosing to focus on a topic that connected to my personal experiences and habits. I already organize my own clothes into categories such as work clothes, workout clothes, dresses, and seasonal outfits, so creating a closet organization website felt natural and easier to build meaningful content around.

Another major decision was to keep the project simple and focused, rather than trying to include too many complicated features. I wanted the website to feel organized, easy to navigate, and visually clean, rather than overcrowded. As a result, I concentrated on enhancing usability, design consistency, and layout instead of incorporating unnecessary features.

I also chose to enhance the visual design between Project 01 and Project 02 by incorporating better spacing, hover effects, cleaner cards, and a more responsive layout. These adjustments made the website appear more polished and professional.

## What worked
The category filtering system was effective and enhanced the site's interactivity. The filter buttons allowed users to quickly sort through different clothing categories. I was pleased with how the JavaScript functionality integrated with the layout, significantly improving the user experience.

I was also satisfied with the overall design improvements. Compared to my original version, the final project feels much more organized and visually balanced. The clothing cards, color palette, and tips section helped make the site more engaging while keeping the design simple.

Another aspect that worked well was using a topic I genuinely understood. Since closet organization is something I already practice in real life, it made it easier to write descriptions and create a logical structure for the website.

## What I would do differently
If I were to start the project over, I would dedicate more time to planning the design and ensuring responsiveness before writing the majority of the code. Initially, I focused primarily on functionality, but later I had to revisit and adjust the spacing, layout, and mobile usability.

I would also test the site more frequently throughout development instead of waiting until later stages. Some bugs, such as issues with buttons and styling, could have been fixed earlier if I had tested more consistently.

Another thing I would do differently is add more advanced features, such as saving favorite outfit categories or adding a search feature. While I am happy with the simplicity of the final project, adding a few more interactive features could improve the overall experience even more.

## What I learned
This project helped me better understand the full web development process, from planning and coding to debugging and deployment. I learned that creating a successful website is not only about making something functional, but also about making it visually appealing and user-friendly.

I learned the importance of iteration throughout my final project. It became significantly stronger because I consistently sought improvements instead of settling for the initial version. Making small adjustments, such as refining spacing, fixing responsiveness, and enhancing styling, greatly contributed to the final outcome.

In addition, I gained more confidence working with HTML, CSS, and JavaScript together. I learned how styling, layout, and interactivity all work together to create a better user experience. Overall, this project taught me the value of organization, planning, and attention to detail when building for the web.
